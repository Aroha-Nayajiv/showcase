# Medical History Schema (Overview)

## Insurance Service Design Specification

### Overview
The InsuranceService provides CRUD operations for patient insurance records, ensuring data confidentiality via field‑level encryption (AES‑256‑GCM) using PostgreSQL pgcrypto. It integrates with AuthService for JWT‑based RBAC and AuditLogService for comprehensive audit trails.

### Data Model
| Entity | Field | Type | Required | Constraints |
|---|---|---|---|---|
| insurance_info | id | UUID PK | Yes | Generated by system |
| insurance_info | patient_id | UUID FK | Yes | References patient.id, indexed |
| insurance_info | provider_name | TEXT | Yes |  |
| insurance_info | policy_number | TEXT | Yes | Encrypted at rest, unique per patient |
| insurance_info | group_number | TEXT | No | Encrypted if present |
| insurance_info | effective_date | DATE | Yes | <= expiration_date |
| insurance_info | expiration_date | DATE | Yes | >= effective_date |
| insurance_info | coverage_type | TEXT CHECK (coverage_type IN ('Medical','Dental','Vision','Pharmacy','Other')) | Yes |  |
| insurance_info | created_at | TIMESTAMPTZ | Yes | DEFAULT now() |
| insurance_info | updated_at | TIMESTAMPTZ | Yes | Updated automatically |

### API Endpoints

#### POST /api/v1/insurance
- Purpose: Create a new insurance record.
- Auth: Bearer token; role must be clinician or front_desk.
- Request JSON Schema:
{
  'patient_id': 'uuid',
  'provider_name': 'string',
  'policy_number': 'string',
  'group_number': 'string?',
  'effective_date': 'date',
  'expiration_date': 'date',
  'coverage_type': 'string'
}
- Response (201):
{
  'id': 'uuid',
  'status': 'created',
  'created_at': 'datetime'
}
- Errors: ERR-001 (400 validation), ERR-002 (401 auth), ERR-003 (403 RBAC), ERR-005 (500 internal).

#### GET /api/v1/insurance/{id}
- Purpose: Retrieve a single insurance record (decrypted).
- Auth: Bearer token; role clinician, front_desk, or admin.
- Response (200):
{
  'id':'uuid',
  'patient_id':'uuid',
  'provider_name':'string',
  'policy_number':'string',
  'group_number':'string?',
  'effective_date':'date',
  'expiration_date':'date',
  'coverage_type':'string',
  'created_at':'datetime',
  'updated_at':'datetime'
}
- Errors: ERR-001 (400 id format), ERR-002 (401), ERR-003 (403), ERR-004 (404 not found), ERR-005 (500).

#### PUT /api/v1/insurance/{id}
- Purpose: Update existing record.
- Auth: Bearer token; role admin or owning clinician.
- Request JSON Schema (at least one field):
{
  'provider_name':'string?',
  'policy_number':'string?',
  'group_number':'string?',
  'effective_date':'date?',
  'expiration_date':'date?',
  'coverage_type':'string?'
}
- Response (200):
{
  'id':'uuid',
  'status':'updated',
  'updated_at':'datetime'
}
- Errors: ERR-001, ERR-002, ERR-003, ERR-004, ERR-005.

#### DELETE /api/v1/insurance/{id}
- Purpose: Delete a record.
- Auth: Bearer token; role admin.
- Response (200):
{
  'status':'deleted'
}
- Errors: ERR-001, ERR-002, ERR-003, ERR-004, ERR-005.

#### GET /api/v1/insurance
- Purpose: List insurance records for a patient with pagination.
- Auth: Bearer token; role clinician or front_desk.
- Query Parameters: patient_id=uuid, page=integer (default=1), page_size=integer (default=20, max=100).
- Response (200):
{
  'page':1,
  'page_size':20,
  'total_items':123,
  'items':[
    {
      'id':'uuid',
      'provider_name':'string',
      'policy_number':'string',
      'coverage_type':'string',
      'effective_date':'date',
      'expiration_date':'date'
    } // ... up to page_size items
  ]
}
- Errors: ERR-001 (invalid query), ERR-002, ERR-003, ERR-005.

### Service Boundaries
| Service | Responsibility | Dependencies | Events Emitted | Events Consumed |
|---|---|---|---|---|
| InsuranceService | CRUD for insurance_info, encrypt/decrypt fields, enforce RBAC, emit audit events | PostgreSQL, AuthService (JWT validation), AuditLogService | InsuranceCreated, InsuranceUpdated, InsuranceDeleted | PatientDeleted (cascade), RoleChanged |

### Security Considerations
- Authentication: JWT signed with RS256, validated by AuthService.
- Authorization: Row‑level security (RLS) policies restrict SELECT/UPDATE/DELETE to records where patient_id matches clinicians' assigned patients; admin role has full access.
- Encryption: policy_number and group_number stored encrypted using pgcrypto AES‑256‑GCM; decryption performed after auth check. Key rotation quarterly per internal KMS policy.
- Audit Logging: Every request logs user_id, action, resource_id, timestamp, and outcome to AuditLogService. Logs retained for 7 years (FR‑003).
- Input Validation: JSON schema validation on all endpoints; dates validated for logical order.
- Rate Limiting: API gateway enforces ≤100 requests/second per client IP.
